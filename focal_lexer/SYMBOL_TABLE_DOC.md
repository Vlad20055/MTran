"""
ДОКУМЕНТАЦИЯ: Таблицы имён (Symbol Tables) в focal_lexer
==========================================================

Описание:
---------
После лексического анализа, информация о некоторых лексемах собирается
и записывается в таблицы имён. Модуль symbol_table.py реализует все
необходимые таблицы для работы с символами.

Структура таблиц:
-----------------

1. ТАБЛИЦА КЛЮЧЕВЫХ СЛОВ (Keywords Table)
   - Содержит все команды языка FOCAL
   - Примеры: C (COMMENT), A (ASK), T (TYPE), S (SET), IF, FOR и т.д.
   - Структура: {'letter': {'name': 'command_name', 'type': 'Ключевое слово'}}

2. ТАБЛИЦА ФУНКЦИЙ (Functions Table)
   - Содержит встроенные математические функции
   - Примеры: FSIN (синус), FCOS (косинус), FABS (модуль) и т.д.
   - Структура: {'NAME': {'name': 'NAME', 'type': 'Функция', 'description': '...'}}

3. ТАБЛИЦА РАЗДЕЛИТЕЛЕЙ (Delimiters Table)
   - Содержит символы-разделители
   - Примеры: , (запятая), ; (точка с запятой), () (скобки), ! (перевод строки)
   - Структура: {'symbol': {'symbol': 'sym', 'type': 'Разделитель', 'name': 'NAME'}}

4. ТАБЛИЦА ОПЕРАТОРОВ (Operators Table)
   - Содержит математические и логические операторы
   - Примеры: + - * / ^ =
   - Структура: {'op': {'symbol': 'op', 'type': 'Оператор', 'name': 'NAME', 'operation': '...'}}

5. ТАБЛИЦА ИДЕНТИФИКАТОРОВ (Identifiers Table) - ДИНАМИЧЕСКАЯ
   - Заполняется во время анализа кода
   - Содержит все переменные, встреченные в исходном коде
   - Номер элемента увеличивается по мере добавления новых идентификаторов
   - Если идентификатор встречается повторно, ему присваивается тот же номер

6. ТАБЛИЦА КОНСТАНТ (Constants Table) - ДИНАМИЧЕСКАЯ
   - Заполняется во время анализа кода
   - Содержит все числовые константы (числа с плавающей точкой)
   - Номер элемента увеличивается по мере добавления новых констант
   - Если константа встречается повторно, ей присваивается тот же номер

Пример использования:
---------------------

from focal_lexer import SymbolTable, LexicalAnalyzer, TokenType

# Создание таблицы символов
symbol_table = SymbolTable()

# Добавление идентификаторов
id1 = symbol_table.add_identifier('COST', 'Переменная с плавающей точкой')
id2 = symbol_table.add_identifier('PRICE', 'Переменная с плавающей точкой')

# Добавление констант
const1 = symbol_table.add_constant('0.98', 'Константа с плавающей точкой')
const2 = symbol_table.add_constant('10.5', 'Константа с плавающей точкой')

# Вывод таблиц
symbol_table.print_identifiers_table()  # Выведет таблицу идентификаторов
symbol_table.print_constants_table()    # Выведет таблицу констант
symbol_table.print_keywords_table()     # Выведет таблицу ключевых слов
symbol_table.print_functions_table()    # Выведет таблицу функций
symbol_table.print_delimiters_table()   # Выведет таблицу разделителей
symbol_table.print_operators_table()    # Выведет таблицу операторов
symbol_table.print_all_tables()         # Выведет все таблицы

# Получение информации
id_info = symbol_table.get_identifier('COST')
const_info = symbol_table.get_constant('0.98')

API Документация:
-----------------

class SymbolTable:
    
    __init__()
        - Инициализирует все таблицы символов
    
    add_identifier(name: str, info: str = "Переменная") -> int
        - Добавляет идентификатор в таблицу
        - Возвращает номер элемента в таблице
        - Если идентификатор уже есть, возвращает его существующий номер
    
    add_constant(value: str, const_type: str = "Константа с плавающей точкой") -> int
        - Добавляет константу в таблицу
        - Возвращает номер элемента в таблице
        - Если константа уже есть, возвращает её существующий номер
    
    get_identifier(name: str) -> dict | None
        - Возвращает информацию об идентификаторе
        - Возвращает None, если идентификатор не найден
    
    get_constant(value: str) -> dict | None
        - Возвращает информацию о константе
        - Возвращает None, если константа не найдена
    
    print_keywords_table()
        - Выводит таблицу ключевых слов в форматированном виде
    
    print_functions_table()
        - Выводит таблицу функций в форматированном виде
    
    print_delimiters_table()
        - Выводит таблицу разделителей в форматированном виде
    
    print_operators_table()
        - Выводит таблицу операторов в форматированном виде
    
    print_identifiers_table()
        - Выводит таблицу идентификаторов в форматированном виде
    
    print_constants_table()
        - Выводит таблицу констант в форматированном виде
    
    print_all_tables()
        - Выводит все таблицы в порядке:
          1. Ключевые слова
          2. Функции
          3. Разделители
          4. Операторы
          5. Идентификаторы
          6. Константы

Файлы модуля:
--------------
- symbol_table.py     - Основной модуль с классом SymbolTable
- __init__.py         - Экспортирует SymbolTable
- main.py             - Пример использования (демонстрирует работу таблиц)

Интеграция с лексером:
-----------------------
Таблицы символов работают совместно с LexicalAnalyzer и Token:

1. LexicalAnalyzer сканирует исходный код и генерирует токены
2. Токены содержат информацию о типе лексемы и её значении
3. SymbolTable собирает информацию из токенов и организует её в таблицы
4. Таблицы можно выводить и использовать для анализа кода

Пример прохода данных:
исходный код → LexicalAnalyzer → Token → SymbolTable → Таблицы
"""
